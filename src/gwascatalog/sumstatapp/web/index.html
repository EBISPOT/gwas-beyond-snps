Welcome to the Beyond SNPs Submission Tool

This tool will check your summary statistics files contain valid data and automatically prepare them for submission.

If you need help with your submission, please email gwas-subs@ebi.ac.uk.

All validation runs locally in your browser. Your data is not uploaded to a server.

If you need to validate more than a few summary statistic files, the command line interface is a better way to get started.

Thank you for submitting your data to the GWAS Catalog.

What kind of genetic variation have you studied?

Why do we ask this?

Different types of genetic variants have different data requirements. For example, structural variants like CNVs require start and end positions (a range), while SNPs only require a single position.

Select variation type

SNP submissions are not supported by this tool yet.

Please see gwas-sumstats-tools for SNP submissions.

Only SNP, CNV, and gene-based genome-wide studies are currently validated by the GWAS Catalog.

To enquire about submitting a non-standard data type or pilot study, please contact gwas-subs@ebi.ac.uk.

Which columns are present in your summary statistics?

Why do we ask this?

Different types of GWAS data are represented in different ways. Telling us what's in your file helps us validate it correctly.

This checklist describes the fields (column names) that should be present in your summary statistics table.

If you're not sure what to put in each field, please download the example file from here and adapt it to your data.

Column names look like this. Column names must appear exactly as shown below. Any differences or typos in your files will cause validation errors.

Gene name

How are genes named in your data?

HGNC symbol (e.g. TP53).
Ensembl gene ID (e.g. ENSG00000141510).

Please select how genes are named in your data.

p value representation

How are p values stored in your file? Please only include one field.

p value (e.g. 0.00034). Smaller p values are more significant.
Negative log10 p value (e.g. 3.47). Larger values indicate greater significance.

Please select how p values are represented in your data.

Position

How is the position of the gene represented?

Chromosome where the gene is located
Start position of the gene (0/1 based)
End position of the gene (0/1 based)

If you include position data, all three fields (chromosome, start, end) are required.

Effect size

Measuring effect sizes is optional for gene-based GWAS. The most common choices are beta and odds ratio.

Regression coefficient.
Odds ratio estimate.
Z-score statistic.
Hazard ratio estimate.

Odds ratio and hazard ratio cannot both be selected as effect sizes.

Primary effect size

You've selected multiple effect sizes. Which is the primary measure of effect for your study?

(The primary measure of effect will be put in a standardised column index in the output file.)

Uncertainty estimate

If you include an effect size, you must provide an uncertainty estimate.

The most common choices are standard error for beta, and confidence intervals for odds ratios or hazard ratios.

Standard error.
Lower bound of the confidence interval (typically odds ratio).
Upper bound of the confidence interval (typically odds ratio).

Standard error is only valid for beta, and beta requires a standard error.

Z-score is a standardised statistic and cannot have an associated uncertainty estimate.

Confidence intervals are only valid for odds ratio or hazard ratio. Both bounds must be provided if one is given.

Are highly significant p values thresholded or rounded to zero?

Why do we ask this?

Thresholding or rounding p values limits the downstream usability of the data.

If your data includes p values thresholded or rounded to zero, please provide negative log₁₀ p values instead.

Are p-values thresholded?

If your data includes p values thresholded or rounded to zero, please provide negative log₁₀ p values instead.
